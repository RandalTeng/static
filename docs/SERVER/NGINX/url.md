Nginx路由匹配规则
==

匹配原型
--

``` nginx conf
location [] /uri/ {

}

"[]" 可能的值:
1. =    : 精确完全匹配, 优先级最高
2. ^~   : 精确前缀匹配, 优先级第二
3. ~    : 区分大小写的正则匹配, 优先级第三
4. ~*   : 不区分大小写的正则匹配
5. /uri : 普通前缀匹配, "[]"为空, 优先级第四
6. /    : 通用匹配, "[]"为空, 可以匹配所有请求, 优先级最低
```

> 所有匹配, 一旦匹配到符合的规则, 不再继续向下匹配  
> 查找顺序, 按优先级排序, 同优先级按配置文件配置顺序
